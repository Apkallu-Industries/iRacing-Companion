## 🚀 Key Improvements & Architectures

### 1. Dynamic Sim-Specific Profiles & Capabilities Matrix
In [vehicleProfiles.cjs](file:///c:/Dev/iRacing-Companion/desktop/telemetry/vehicleProfiles.cjs), we introduced:
* **Simulator Anchoring:** All profiles are explicitly tagged (e.g., `simulator: "iRacing"`).
* **New Vehicle Classes:** Add modular parameters for **Rallycross** (very high slip, quick steering, bottoming vel threshold = 0.25) and **Oval** (stability lag, high aeromechanic minimum speed = 160 KPH).
* **Vehicle Dynamics Capability Matrix:** An explicit `capabilities` mapping determines whether a car supports key telemetry detections:
  ```javascript
  capabilities: {
    aeroStallDetect: "None" | "Medium" | "High" | "Very High" | "Extreme",
    brakeLockDetect: "None" | "Low" | "Medium" | "High" | "Very High",
    diffInstability: "None" | "Medium" | "High" | "Very High",
    rideHeightSensitivity: "None" | "Low" | "Medium" | "High" | "Extreme"
  }
  ```
  If a capability is `"None"`, the detector engine bypasses calculations to prevent invalid anomalies.

### 2. Enriched Telemetry Provenance
In [coreSchema.cjs](file:///c:/Dev/iRacing-Companion/desktop/telemetry/coreSchema.cjs), [normalizeTelemetry.cjs](file:///c:/Dev/iRacing-Companion/desktop/telemetry/normalizeTelemetry.cjs), and [physicsDerivations.cjs](file:///c:/Dev/iRacing-Companion/desktop/telemetry/physicsDerivations.cjs), we augmented the standard `TelemetryValue` schema to capture detailed data origins:
```json
{
  "value": 0.42,
  "confidence": 0.95,
  "source": "derived",
  "provenance": {
    "derivedFrom": ["inputs.steering"],
    "simSource": null,
    "origin": "iRacing",
    "tickSource": 101,
    "latency": 4
  }
}
```
* **latency:** Dynamically calculated at 60Hz as `Date.now() - timestamp` in milliseconds to detect network/ingestion lag.
* **tickSource:** Records the raw frame tick count of the original acquisition.

### 3. Class-Aware Contextual Detectors
In [eventDetector.cjs](file:///c:/Dev/iRacing-Companion/desktop/telemetry/eventDetector.cjs), we removed generic locking thresholds and replaced them with separate class-specific sub-detector routines:
* **`LockupDetectorGT3`:** Moderates lockups with ABS rules (severity: `WARNING`, risk: `4`).
* **`LockupDetectorNASCAR`:** Severe flat-spot risks without ABS (severity: `CRITICAL`, risk: `7`).
* **`LockupDetectorLMP2`:** Deceleration aero load drop lockups (severity: `CRITICAL`, risk: `6`).
* **Capabilities bypass:** If `capabilities.brakeLockDetect === "None"` (e.g. Rallycross), the lockup detector immediately returns empty arrays.

### 4. Grounded Physics Reasoning Layer
In [mechanicalReasoningEngine.ts](file:///c:/Dev/iRacing-Companion/src/lib/session-intelligence/mechanicalReasoningEngine.ts), we:
* Enhanced `inferMechanicalState` to check for trail-braking over-rotation snaps (`ENTRY_OVER_ROTATION`), classifying them semantically as `"LIFT_OFF_OVERSTEER_BRAKE_RELEASE_TIMING"`.
* Implemented the **AI Strategist Gatekeeper Filter** (`getStrategistMechanicalState`), which blocks all semantic inferences below a strict **`70%` confidence threshold** to guarantee the strategist co-driver only reasons on physically sound motorsport realities.
