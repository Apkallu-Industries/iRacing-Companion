## 2. Key Architectural Upgrades

### A. Extended CoreTelemetryV1 Schema Contract
Added two strict provenance channels of type `TelemetryValue<string>` to [coreSchema.cjs](file:///c:/Dev/iRacing-Companion/desktop/telemetry/coreSchema.cjs):
* `physicsProfile`: Models the target physics assumptions container (e.g. `GT3_iRacing`).
* `physicsProfileVersion`: Tracks version modifications of dynamic parameters to ensure historical reproducibility.

* **SF23 / F4 / Oval**: v1.0.0
* **Rallycross**: `Rallycross_iRacing` v1.1.0 (with loose surface suppression)

### C. Stateless Normalized Acquisition
Modified [normalizeTelemetry.cjs](file:///c:/Dev/iRacing-Companion/desktop/telemetry/normalizeTelemetry.cjs) to resolve the active vehicle profile during normalization:
* Enriches the normalized frame with strict `TelemetryValue` wrappers containing metadata (measured source, latency, tick source).
* Standardized clean fallbacks in `createDefaultFrame()` to preserve system stability.

### D. Deep Provenance Event Serialization
Updated [eventDetector.cjs](file:///c:/Dev/iRacing-Companion/desktop/telemetry/eventDetector.cjs) to automatically serialize active vehicle dynamic profiles:
* The standard event factory `createTelemetryEvent` now extracts `physicsProfile` and `physicsProfileVersion` from the resolved context.
* Binds these parameters permanently onto every emitted `TelemetryEvent` object.

### E. Unified TypeScript and Schema Taxonomy
* Updated [eventTaxonomy.ts](file:///c:/Dev/iRacing-Companion/src/lib/session-intelligence/eventTaxonomy.ts) to define `physicsProfile` and `physicsProfileVersion` within the TypeScript `TelemetryEvent` contract.
