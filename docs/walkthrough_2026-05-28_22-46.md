# Walkthrough — Telemetry Ingestion & Normalization Layer (v1)

**Date:** 2026-05-28  
**Time:** 22:46:00 (UTC)

We have successfully implemented the first stable telemetry normalization and in-process routing layer. The supervisor is now fully equipped to act as a real-time sim data router, strictly enforcing `CoreTelemetryV1` for standard consumers.

---

## 🛠️ Changes Completed

### 1. Canonical Schema & Normalizer

- Created `coreSchema.cjs` to define the strict `CoreTelemetryV1` structure (covering timestamps, speed, RPM, gear, core inputs, and lap timing).
- Created `normalizeTelemetry.cjs`, a pure, stateless mapper that filters all raw simulator inputs, performs conversions (e.g. `speedKph` to `m/s`, `steeringDeg` to radians), and safely discards any simulator-specific auxiliary tags (Strict Mapping).

### 2. Ingestion & Dual-Emission

- Modified `ingestionController.cjs` to process telemetry packets through standard asynchronous queues, applying a `MAX_QUEUE_LENGTH` backpressure guard.
- Added dual routing endpoints (`onTelemetryPacket` for normalized frames and `onRawTelemetryPacket` for raw packets) allowing consumers to pick the exact dataset appropriate for their use case.

### 3. Supervisor & API Integration

- Modified `runtimeSupervisor.cjs` to dynamically start and stop the local UDP/TCP ingestion streams bound directly to `startSession` and `stopSession` commands.
- Extended the HTTP controller in `supervisorApi.cjs` to host distinct Server-Sent Events (SSE) connections:
  - `GET /telemetry/live`: Emits strict normalized `CoreTelemetryV1` objects.
  - `GET /telemetry/raw`: Emits unfiltered raw sim packets.
  - `GET /telemetry/schema`: Returns version and structure definitions.

### 4. IPC & Preload Interfacing

- Modified `main.cjs` to subscribe to supervisor streams in-process and dispatch telemetry down standard IPC channels to Electron browser windows.
- Exposed these events securely to front-end components inside `preload.cjs` via:
  - `window.pitWallRuntime.onTelemetryLive(callback)`
  - `window.pitWallRuntime.onTelemetryRaw(callback)`
  - `window.pitWallRuntime.getTelemetrySchema()`
