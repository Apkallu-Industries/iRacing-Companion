* Re-engineered [eventDetector.cjs](file:///c:/Dev/iRacing-Companion/desktop/telemetry/eventDetector.cjs) to route all enqueued rules through **Physical Plausibility Gates**:
  * **Speed Gate:** Aerodynamic platform bottoming is strictly rejected if body speed falls below the class threshold (e.g. 120-180 KPH), eliminating false alerts during low-speed curbs or pit garages.
  * **Brake Lock Gate:** Lockup alerts are rejected if speed is $< 5$ KPH or brake input is $< 10\%$.
  * **Entry Snap Gate:** Snap oversteer alerts are rejected if steering lock is straight-line ($< 0.02$ radians) to filter out wind gusts.
  * **Wheelspin Gate:** Traction decay events are rejected if throttle input is $< 10\%$.

### 4. Dynamic Confidence Calculations
* Rewrote [physicsDerivations.cjs](file:///c:/Dev/iRacing-Companion/desktop/telemetry/physicsDerivations.cjs) to estimate live model confidence:
  * expectedYawRate confidence decays at low speeds where tire friction dominates over kinematics.
  * understeer/oversteer index confidence decays under micro-steering angles where signal noise is prominent.
  * Gradients decay under high frame-tick latency jitter.
