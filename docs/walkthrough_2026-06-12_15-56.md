# Walkthrough — Electron to Tauri Migration and Native Rust Telemetry Reader

The legacy Electron wrapper has been completely removed in favor of Tauri, and the telemetry shared-memory reader for Assetto Corsa has been rewritten natively in Rust.

## Changes Made

### Workspace & Boot Configuration
- **[package.json](file:///c:/Dev/iRacing-Companion/package.json)**: Removed the `"desktop"` workspace and clean deleted all Electron-specific scripts.
- **[start-all.bat](file:///c:/Dev/iRacing-Companion/start-all.bat)**: Simplified the startup script to directly run Vite, the Local Telemetry Bridge, and Tauri Developer Shell (`npm run tauri:dev`) in parallel.
- **[desktop](file:///c:/Dev/iRacing-Companion/desktop)**: Deleted the entire directory containing Electron main/preload/supervisor files.

### Telemetry Bridge TCP Ingestion
- **[local-bridge/server.js](file:///c:/Dev/iRacing-Companion/local-bridge/server.js)**: Replaced child-process spawning of `ac-reader.exe` with an active TCP server listening on `127.0.0.1:4712`. Received lines are parsed as JSON frames and routed exactly like standard out did previously.
- **[ac-reader.exe / ac-reader.cs](file:///c:/Dev/iRacing-Companion/local-bridge/)**: Clean deleted the C# wrapper source and compiled executable.

### Native Rust Memory Mapping Reader in Tauri
- **[ac_reader.rs](file:///c:/Dev/iRacing-Companion/src-tauri/src/ac_reader.rs)**: Created a native Rust module that performs:
  - Memory-mapped file FFI calls to `"Local\\acpmf_physics"`, `"Local\\acpmf_graphics"`, and `"Local\\acpmf_static"`.
  - Type-safe mappings matching the memory alignment constraints of the game's telemetry structs.
  - Periodic background polling (~60Hz tick intervals) and UTF-16 string normalization.
  - Telemetry streaming over a persistent TCP socket to `127.0.0.1:4712`.
- **[lib.rs](file:///c:/Dev/iRacing-Companion/src-tauri/src/lib.rs)**: Hooked `ac_reader::start_ac_reader_thread()` into Tauri's native setup block.

## Validation Results

- **Tauri Rust Compilation**: Executed `cargo check` inside `src-tauri`, which compiled successfully with zero errors/warnings.
- **Bridge Syntax Verification**: Executed `node --check local-bridge/server.js`, confirming that the TCP ingestion edits are syntactically valid Javascript.
- **Launch Script dry-run**: Verified that the startup runner directly launches Vite and the Node.js bridge in the background, then boots Tauri without any legacy prompts.
