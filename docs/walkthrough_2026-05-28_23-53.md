## 🛠️ Changes Completed

### 1. Higher-Order Derived Physics Engine

- Created [physicsDerivations.cjs](file:///c:/Dev/iRacing-Companion/desktop/telemetry/physicsDerivations.cjs), a stateless math engine that:
  - Computes the **Expected Yaw Rate** using a kinematic bicycle model based on wheelbase metrics and steering geometry.
  - Estimates **Understeer** and **Oversteer Indices** by comparing expected kinematic paths against actual yaw sensor rate inputs.
  - Calculates real-time **Steering Velocity** and **Input Aggression Gradients** (throttle/brake rates of change).
  - Implements a fallback physical wheelbase lookup database based on the active `car` name (e.g. Dallara P217, etc.).

### 2. Sliding Temporal Ring Buffer

- Integrated a sliding memory queue `TelemetryRingBuffer` directly inside [ingestionController.cjs](file:///c:/Dev/iRacing-Companion/desktop/ingestionController.cjs) that retains a continuous, rolling **30-second window** (1800 frames at 60Hz) of normalized frames to provide contextual temporal history.

### 3. Temporal Semantic Event Detection Rules Engine

- Created [eventDetector.cjs](file:///c:/Dev/iRacing-Companion/desktop/telemetry/eventDetector.cjs), a high-frequency rules engine that continuously scans the temporal window to trigger canonical motorsport events:
  - **`BRAKE_LOCK_FRONT_LEFT` / `BRAKE_LOCK_FRONT_RIGHT`**: Triggered when brake pressure is high (>40%) and individual wheel speed decays `< 85%` of vehicle velocity.
  - **`REAR_TRACTION_COLLAPSE` (Wheelspin)**: Triggered when rear wheel velocity exceeds chassis velocity by `> 12%` under positive throttle.
  - **`AERO_BOTTOM_OUT`**: Triggered by high vertical shock compression velocity (>0.15 m/s) at high speeds.
  - **`ENTRY_OVER_ROTATION` (Snap Oversteer)**: Triggered when actual yaw rate exceeds kinematically expected turning rate on turn-in under deceleration.
  - **`EXIT_UNDERSTEER` (Push)**: Triggered when actual yaw rate falls wide of kinematically expected turning rate under positive throttle gradient on exit.
- Implements a **cooldown guard** (1.5 seconds) to prevent event-bus flooding or sensor flickering under rapid fluctuations.

### 4. Supervisor & Stream Infrastructure

- Wired dual streams in [runtimeSupervisor.cjs](file:///c:/Dev/iRacing-Companion/desktop/runtimeSupervisor.cjs) to forward semantic events via the `onEvent` hook.
- Extended [supervisorApi.cjs](file:///c:/Dev/iRacing-Companion/desktop/supervisorApi.cjs) to host a new SSE endpoint `/telemetry/events` that broadcasts event structures to subscribed scripts, mobile devices, and overlays.
- Interfaced IPC handlers inside [main.cjs](file:///c:/Dev/iRacing-Companion/desktop/main.cjs) and securely exposed them in the Electron contextBridge layer [preload.cjs](file:///c:/Dev/iRacing-Companion/desktop/preload.cjs) via:
