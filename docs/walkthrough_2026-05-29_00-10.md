## 🚀 Architectural Extensions

### 1. Dynamic Detector Versioning & Auditing
Every emitted event now explicitly publishes the precise algorithm routine and semantic model version that triggered it:
```json
{
  "eventType": "BRAKE_LOCK_FRONT_LEFT",
  "detector": "LockupDetectorGT3",
  "detectorVersion": "1.1.0"

```
This is a massive addition for:
* **AI Training Reproducibility:** Allows ML layers to audit exact feature definitions over historical sessions.
* **Replay Compatibility & Forensic Auditing:** Guarantees backward-compatible telemetry parsing as physical models evolve.
* **Sub-Detector Isolation:** Maps specific versioned locks across cars, distinguishing `LockupDetectorGT3` (ABS-assisted model v1.1.0) from `LockupDetectorNASCAR` (mechanical friction model v1.1.0) and `LockupDetectorRallycross` (duration-checked model v1.1.0).

### 2. Surface-Aware Rallycross Observability (INFO Tier)
Instead of fully disabling brake lock detection on Rallycross (which creates severe dynamic blind spots), we implemented a surface-aware detector:
* **Observability Gate:** `capabilities.brakeLockDetect` is set to `"INFO"` to allow detection on asphalt transitions while suppressing false alarms on gravel.
* **Surface-Aware Suppression:** The lockup engine checks the raw iRacing `PlayerTrackSurfaceMaterial` telemetry variable. If the car is on dirt (`4`), gravel (`5`), sand (`6`), or grass (`3`), lockup warnings are suppressed.
* **O(1) Duration Filter (220ms):** Emits locking events on asphalt *only* if the wheels remain locked for a consecutive threshold of $\ge 220$ms (inferred dynamically using session timestamps).
* **Slip Threshold (0.45):** Allows wide lateral sliding and heavy transient wheel loads, only flagging locks when wheel speed decays below 45% of body velocity.

### 3. Compound Source-Weighted Confidence
Not all confidence parameters are equal. We introduced a mathematical trust compounder (`getSourceWeightedConfidence`) that weights base heuristical confidence against dynamic mathematical bounds:
* **Direct measured telemetry:** `0.95` trust weight
* **Deterministic derivations:** `0.90` trust weight
* **Behavioral / Probabilistic models:** `0.70` trust weight
* **AI Semantic opinion:** `0.50` trust weight

Compounded confidence is calculated as `baseConfidence * compoundWeight` (taking the minimum trust weight of all active source types), yielding highly credible math context. For example, a Rallycross locking event on asphalt has a base confidence of `0.65` multiplied by `0.95` (deterministic physics), resulting in a perfect compounded confidence of `0.6175`.

### 4. Real-Time Temporal Confidence Decay
Derived telemetry metrics and generated event confidences decay dynamically over time if packets stall or experience significant latency gaps:
* **鮮度 (Freshness) Gate:** If the raw packet's ingestion latency (`freshnessMs = Date.now() - timestamp`) exceeds **`100`ms** (roughly 6 frames at 60Hz), derived actual/expected yaw rates and dynamic event confidences degrade exponentially.
* **Decay Model:** Decays derived confidence linearly to `0` over `1.0` second of stalled updates, ensuring downstream systems ignore stale telemetry during pause states or packet drops.

### 5. "Observed" vs "Interpreted" Event Classifications
We introduced a strict `classification` attribute to all generated events to separate pure sensory measurements from cognitive deductions:
* **`observed` (Direct Physical States):** Lockups (`BRAKE_LOCK_FRONT_LEFT`), wheel spins (`REAR_TRACTION_COLLAPSE`), underbody grounding (`AERO_BOTTOM_OUT`).
* **`interpreted` (Behavioral / Mechanical Tradeoffs):** Decel Snap turn-in rotates (`ENTRY_OVER_ROTATION`), exit lateral pushes (`EXIT_UNDERSTEER`).

### 6. Tiered AI Reasoning Gatekeepers
In [mechanicalReasoningEngine.ts](file:///c:/Dev/iRacing-Companion/src/lib/session-intelligence/mechanicalReasoningEngine.ts), we replaced the single hard confidence cutoff with three structured gatekeeper functions:
* **`getStrategistMechanicalState`:** Minimum confidence gate of **`90%`** (Strategist-safe to prevent tactical hallucinations).
* **`getCoachingMechanicalState`:** Minimum confidence gate of **`75%`** (Coaching-safe for dynamic probabilistic driver tuition).
* **`getOverlayMechanicalState`:** Minimum confidence gate of **`60%`** (Overlay-safe for visual HUD dials).

### 7. "Detector Rationale" Explainability
Every telemetry event now publishes a strict `rationale` array explaining exactly why it triggered (e.g., `["wheel_speed_decayed_below_slip_threshold", "minimum_duration_exceeded", "asphalt_surface_transition"]`). This provides complete transparency for visual charts, replay analysis, and strategist context.
